# iPAS 資安工程師初級考古題 110-1 錯題檢討

## 資訊安全技術概論
**下列何種應用服務會使用到傳輸控制協定（Transmission Control Protocol, TCP）和使用者資料報協定（User Datagram Protocol, UDP）兩種傳輸層協定？**  
(A) FTP (B) SMTP (C) Telnet ***(D) DNS***  

關鍵判斷：  
- `FTP`、`SMTP`、`Telnet` 主要使用 `TCP`；`DNS` 則通常使用 `UDP 53` 查詢，但在區域傳送或回應資料過大等情況下會使用 `TCP 53`。

判斷規則：  
- `FTP` → TCP
- `SMTP` → TCP
- `Telnet` → TCP
- `DNS` → UDP 53 為主，但也會用 TCP 53

---
**關於傳輸控制協定（Transmission Control Protocol, TCP）連接狀態，下列敘述何者「不」正確？**  
(A) SYN-SENT：Client 送往 Server 的第一個要求連線封包（已送出 SYN，在等待對方回覆的期間）   
(B) SYN-RECEIVED：Server 收到 Client 的連線後，回覆給 Client SYN+ACK，並且等待 Client 回覆 ACK  
(C) ESTABLISHED：Client 回覆 ACK 後，Client 及 Server 均進入 Established 狀態，雙方可以正常傳輸，TCP 三向交握（Three-way Handshake）已完成  
***(D) CLOSE-WAIT：被動關閉方收到 FIN 後發出 SYN+ACK 時，會進入這個狀態，此時可以傳輸資料，也可以接收資料***  

關鍵判斷：  
- `SYN+ACK` 屬於 TCP 三向交握建立連線時的封包，不是關閉連線時使用的封包。
- `CLOSE-WAIT` 是被動關閉方收到對方的 `FIN` 後，回覆 `ACK`，並等待本機應用程式關閉連線的狀態。

判斷規則：  
- `SYN`、`SYN+ACK`、`ACK` → 建立連線，三向交握。  
- `FIN`、`ACK` → 關閉連線。
- `CLOSE-WAIT` → 被動關閉方已收到 `FIN` 並回 `ACK`，等待本機程式關閉。

**下列何者「不」是整合式威脅管理（Unified Threat Management, UTM）設備的常見功能？**  
(A) 啟用入侵防禦系統（Intrusion Prevention System, IPS），防禦異常的網路攻擊封包  
(B) 管理外網與內網的各子網段交換路由  
(C) 以 IPSec 通訊協定建立加密傳輸路由  
***(D) 使用 Honeypot 誘捕系統功能來蒐集與分析入侵威脅***  

關鍵判斷：  
- `UTM` 是把多種網路安全功能整合在同一台設備或平台上的整合式威脅管理設備，常見功能包含防火牆、IPS、VPN、內容過濾、防毒、路由與 NAT 等。
- `Honeypot` 是誘捕攻擊者、蒐集攻擊行為與威脅情資的系統，通常不是 UTM 設備的常見功能。

判斷規則：  
- 看到 `UTM` → 想到防火牆 + IPS + VPN + 防毒 + 內容過濾 + 路由 / NAT。
- 看到 `Honeypot` → 誘捕攻擊者、蒐集攻擊樣態，不是一般 UTM 常見功能。

**關於入侵偵測系統（Intrusion Detection System, IDS），下列敘述何者「不」正確？**  
(A) 扮演良好的偵測角色  
***(B) 為組織內關鍵阻擋之設備***  
(C) 正確的運用可掌握組織內目前資安樣態  
(D) 透過邊界路由以偵測模式（Sniffer Mode）部署並觀察全域流量為最佳部署方式  

關鍵判斷：  
- `IDS` 是入侵偵測系統，主要功能是偵測、告警與記錄可疑活動，不是用來主動阻擋攻擊流量的設備。
- 若要主動阻擋或攔截攻擊流量，較接近 `IPS`、防火牆或 `WAF` 的角色。

選項解析：  
- (B) IDS 不是關鍵阻擋設備，錯誤。
- (C) IDS 若部署與調校得當，可協助掌握組織內異常流量、攻擊嘗試與資安狀況，勉強正確。

判斷規則：  
- `IDS` → 偵測、告警、記錄，不主動阻擋。  
- `IPS` → 預防、攔截、阻擋。

**某網站在設計時進行過安全分析，也在開發時要求程序員編寫安全的代碼，但佈署時由於管理員將備份存放在WEB目錄下導致了攻擊者可直接下載備份，為了發現系統中是否存在其他類似問題，下列何種測試方式是最佳的測試方法？**  
(A) 模糊測試 (B) 源碼測試 (C) 整合測試 ***(D) 滲透測試***  

關鍵判斷：  
本題描述的問題是部署後的環境設定錯誤：管理員把備份檔放在 Web 目錄下，導致攻擊者可直接下載。這類問題不一定能從源碼或一般整合測試中發現，較適合透過滲透測試從攻擊者角度檢查網站是否存在敏感檔案暴露、目錄設定錯誤或部署不當等問題。  

判斷規則：  
- `模糊測試`：大量輸入異常資料，觀察程式是否崩潰或產生異常，偏輸入處理與未知錯誤探索。
- `源碼測試`：檢查程式原始碼問題。但本題問題在部署環境，不在程式碼。
- `整合測試`：測試各模組整合後功能是否正常，偏功能與系統流程，不是最適合找攻擊者可利用的部署弱點。
- `滲透測試`：模擬攻擊者檢查系統實際可被利用的弱點，最符合本題。

**下列何者「不」是進行中間人攻擊（Man-In-The-Middle Attack, MITM）所採用的技巧？**
***(A) Clickjacking*** (B) Sniffing (C) Packet Injecting (D) Session Hijacking  

關鍵判斷：  
`MITM` 中間人攻擊的核心是攻擊者介入通訊雙方之間，攔截、監聽、竄改或接管傳輸內容。
- `Clickjacking`：透過透明 iframe 或偽裝介面誘導使用者點擊，不是 MITM 常用技巧。
- `Sniffing`：封包監聽，符合 MITM。
- `Packet Injecting`：封包注入，符合 MITM。
- `Session Hijacking`：工作階段劫持，可搭配 MITM 取得或利用 session，符合 MITM。  

判斷規則：  
- 看到 `Sniffing`、`Packet Injecting`、`Session Hijacking` → 可聯想到 MITM。
- 看到 `Clickjacking` → UI 誘騙／點擊劫持，不是中間人攻擊。

**關於連線劫持（Session Hijacking）的偵測與防禦，下列敘述何者「不」正確？**
(A) 實踐自動登出的功能也是連線劫持的其中一種防禦方式  
(B) 防毒軟體對連線劫持也有部分功效  
(C) 加密的連線可預防連線劫持  
***(D) IDS可偵測與防禦連線劫持***  

關鍵判斷：  
`IDS` 是入侵偵測系統，主要功能是偵測、告警與記錄可疑行為，不是主動阻擋或防禦攻擊的設備。若要主動阻擋，較接近 `IPS`、防火牆、`WAF` 或其他防護機制。

選項解析：
- (A) 自動登出可縮短 session 有效時間，降低 session 被盜用後可利用的時間，屬於防禦方式。
- (B) 防毒軟體若能偵測**木馬、惡意程式或竊取 cookie/token 的工具**，對連線劫持有部分防護效果。
- (C) 加密連線可降低 session token 被竊聽的風險，因此有助於預防連線劫持。
- (D) IDS 可偵測可疑行為，但**不負責主動防禦或阻擋**，因此本選項不正確。

判斷規則：
- 看到「IDS 可偵測與防禦」→ 小心，通常錯在「防禦」。

**關於運用主機卡模擬（Host Card Emulation, HCE）於行動裝置上進行行動支付，下列敘述哪些是正確的？(1)從雲端支付平台取得的金鑰是有時效性的、(2)無需挑選通過服務平台安全認證的手機、(3)手機無需具備安全元件來儲存支付資訊、(4)需更換具備安全防護特殊的SIM卡才能支援**  
***(A) (1)(2)(3)*** (B) (1)(2)(4) (C) (1)(3)(4) (D) (2)(3)(4)  

關鍵判斷：  
`HCE` 是主機卡模擬，讓行動裝置可用軟體方式模擬卡片，搭配雲端平台提供具時效性的金鑰或 token。其特點是不必依賴手機內建安全元件，也不需要更換具特殊安全防護的 SIM 卡。

判斷規則：  
- `HCE` → 雲端 token / 暫時金鑰
- `HCE` → 不需 Secure Element
- `HCE` → 不需特殊 SIM
- (2)(3) 不能搭 (4)

## 資訊安全管理概論
**下列何者是我國個人資料保護法與歐盟一般資料保護規範（General Data Protection Regulation, GDPR）有較相同的規定？**  
***(A) 資料刪除權***  
(B) 資料可攜權  
(C) 引進新科技之個資處理方式時需進行資料保護影響評估  
(D) 資料保護長制度  

關鍵判斷：  
我國《個人資料保護法》與 GDPR 都有類似「資料刪除」的規定。
- 我國個資法第 3 條、第 11 條規定，當特定目的消失或期限屆滿等情況，當事人得請求刪除、停止蒐集處理或利用個人資料。
- `資料可攜權`、`資料保護影響評估 DPIA`、`資料保護長 DPO` 都是 GDPR 較明確、較具代表性的制度，我國現行個資法並沒有同等完整規範。

---
**關於資訊資產之擁有、保管與使用，下列敘述何者正確？**  
(A) 使用者負責獲得適當的授權，得以存取或異動資訊資產  
(B) 擁有者對於資訊資產負有管理的責任，通常由各使用者擔任  
(C) 使用者負責資訊資產的相關處理與保管工作  
***(D) 資訊資產之擁有、保管與使用的權責，應由各部門權責主管指定適當之擁有者、保管者與使用者*** 

觀念釐清：
資訊資產管理要先明確指定角色與權責，包括：  
- `擁有者`：對資訊資產負管理責任，通常是業務或部門權責主管，不是一般使用者。
- `保管者`：負責資訊資產的保存、維護、處理或技術管理。
- `使用者`：在授權範圍內使用資訊資產。

選項解析：  
- (A) 的陷阱在**異動**資訊資產，使用者沒這權限  
- (B) 使用者不一定是擁有者
- (C) 保管者才負責保管

**關於區塊加密法（Block Cipher），下列敘述何者「不」正確？**
(A) 區塊加密採用多輪運算（Iteration）  
***(B) 多輪運算中，每輪使用的金鑰都是主金鑰***  
(C) 金鑰長度不一定要跟區塊大小相同  
(D) 加密演算法與解密演算法應互為反函式  

關鍵判斷：  
區塊加密法通常會將明文切成固定大小的區塊，並透過多輪運算進行加密。  
多輪運算中，每一輪通常使用由主金鑰透過金鑰排程產生的**子金鑰**，而不是每輪都直接使用同一把主金鑰。  

判斷規則：  
- 區塊加密 → 固定區塊、多輪運算。
- 多輪運算 → 主金鑰透過 key schedule 產生各輪子金鑰。
- 金鑰長度 ≠ 一定等於區塊大小。
- 加密／解密 → 必須可逆。

**關於營運持續計畫（Business Continuity Plan, BCP）之目的，下列敘述何者較「不」正確？**  
***(A) 營運持續計畫是一套基於業務運行規律的管理要求和規範流程***  
(B) 業務持續性是指企業有應對風險、自動調整和快速反應的能力  
(C) 將災難造成的影響減少至最小程度，確保業務流程能及時地繼續運行  
(D) 制訂和實施應變計畫，確保在要求的時間內恢復業務流程  

關鍵判斷：  
- `BCP` 的重點不是日常業務流程管理，而是當災難、事故或重大中斷發生時，降低衝擊，並確保關鍵業務能在要求時間內維持或恢復。
- (A) 太偏日常管理流程，沒有凸顯「中斷時的應變與復原」，因此較不正確。

判斷規則：  
- `BCP` → 出事時撐住營運。
- 關鍵字：災難、中斷、應變、恢復、要求時間內、影響最小化。
